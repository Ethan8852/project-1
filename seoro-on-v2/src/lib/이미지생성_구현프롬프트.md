# 서로ON 이미지 생성 파이프라인 — Antigravity IDE Agent 프롬프트

> Antigravity IDE의 오른쪽 Agent 패널에 아래 프롬프트를 붙여넣으세요.
> 한 번에 전체를 요청하지 말고, 단계별로 나눠서 진행합니다.

---

## 사전 준비 (코드 작성 전 확인)

아래 두 가지 API 키가 있어야 합니다.
- **OpenAI API 키** — 장면 추출(GPT-4o) + 이미지 생성(DALL-E 3)
- **Supabase 프로젝트 URL + anon key** — 이야기 텍스트 저장·불러오기

키가 없으면 먼저 발급 후 `.env` 파일에 아래처럼 넣어두세요.
```
OPENAI_API_KEY=sk-...
SUPABASE_URL=https://xxxx.supabase.co
SUPABASE_ANON_KEY=eyJ...
```

---

## 1단계 — 프로젝트 기본 셋업

```
아래 조건으로 서로ON 이미지 생성 파이프라인의 기본 프로젝트를 셋업해줘.

- 프레임워크: Next.js (App Router)
- 언어: TypeScript
- 패키지: openai, @supabase/supabase-js, dotenv
- .env.local 파일에 OPENAI_API_KEY, SUPABASE_URL, SUPABASE_ANON_KEY 를 읽도록 설정
- 기본 폴더 구조:
  /app
    /api
      /extract-scene     ← 장면 추출 API
      /generate-image    ← 이미지 생성 API
      /story-image       ← 두 단계를 한 번에 처리하는 통합 API
  /lib
    openai.ts            ← OpenAI 클라이언트
    supabase.ts          ← Supabase 클라이언트
    prompts.ts           ← 고정 스타일 블록 상수

설치 명령과 기본 파일들을 만들어줘.
```

## 2단계 — 고정 스타일 블록 상수 등록

```
/lib/prompts.ts 파일을 아래 내용으로 만들어줘.

STYLE_BLOCK 상수에 아래 텍스트를 정확히 그대로 저장할 것.
한 글자도 수정하지 말 것.

---
Soft hand-painted storybook illustration, gentle watercolor and gouache textures with a light paper grain. Warm, nostalgic atmosphere bathed in soft golden natural light with tender, low-contrast shading. Cozy muted palette of warm amber, cream, soft sky blue and gentle sage green. Rounded, soft outlines; simple uncluttered composition with comfortable negative space and one clear focal point. Kind, warm Korean characters shown gently from a slight distance or three-quarter view; peaceful, heartwarming mood. Painterly, hand-drawn animation-still feel — calm, wholesome, easy and comfortable for all ages including the elderly.
Avoid: any text, letters, numbers, logos or watermark; photorealism; harsh shadows; neon or oversaturated colors; cluttered or busy backgrounds; distorted faces or hands.
---

그리고 buildImagePrompt(scene: string): string 함수도 만들어줘.
scene을 받아서 아래 형식의 완성된 프롬프트 문자열을 반환:

[STYLE_BLOCK]
Scene: [scene]
Aspect ratio 4:3, centered storybook framing.
```

## 3단계 — 장면 추출 API

```
/app/api/extract-scene/route.ts 를 만들어줘.

POST 요청으로 { story: string } 을 받아서,
GPT-4o로 이야기에서 핵심 장면 한 문장을 영어로 추출해 반환하는 API.

GPT-4o에 넘길 시스템 프롬프트:
"You are a visual scene extractor for a Korean family memoir illustration service.
Extract one vivid, paintable scene from the given Korean story text.
Output exactly ONE English sentence describing: who (1-2 people), where (Korean setting with era hint if possible), what they are doing, and the lighting/season.
Do NOT include any text, letters, numbers, or words to be shown in the illustration.
Do NOT include any explanation — output the scene sentence only."

유저 메시지: story 내용 그대로.

응답: { scene: string }
에러 처리 포함.
```

---

## 4단계 — 이미지 생성 API

```
/app/api/generate-image/route.ts 를 만들어줘.

POST 요청으로 { scene: string } 을 받아서,
/lib/prompts.ts 의 buildImagePrompt(scene) 로 완성된 프롬프트를 만들고
DALL-E 3 API로 이미지를 생성해 URL을 반환하는 API.

DALL-E 3 설정:
- model: "dall-e-3"
- size: "1792x1024"   ← 4:3에 가장 가까운 옵션
- quality: "standard"
- n: 1

응답: { imageUrl: string }
에러 처리 포함.
```

---

## 5단계 — 통합 API (이야기 → 이미지 한 번에)

```
/app/api/story-image/route.ts 를 만들어줘.

POST 요청으로 { story: string, storyId?: string } 을 받아서
아래 순서로 처리하는 통합 API:

1. GPT-4o로 story에서 scene 한 문장 추출 (3단계 로직 재사용)
2. buildImagePrompt(scene)으로 완성 프롬프트 조합
3. DALL-E 3로 이미지 생성
4. storyId가 있으면 Supabase의 stories 테이블에
   image_url, image_scene, image_generated_at 컬럼 업데이트
5. 응답: { imageUrl: string, scene: string }

각 단계 실패 시 명확한 에러 메시지 반환.
```

## 6단계 — 프론트엔드 연결 (버튼 → 이미지 표시)

```
서로ON 앱의 이야기 상세 화면에 이미지 생성 버튼을 추가해줘.

[이야기 카드 하단에 두 개의 버튼]
- [이미지 생성하기] — /api/story-image 를 호출해 이미지만 표시
- [이야기와 함께 보기] — 이미지 + 정리된 이야기 텍스트를 나란히 표시

상태 관리:
- idle: 버튼만 보임
- loading: "일러스트를 그리고 있어요..." 로딩 표시 (스피너)
- success: 생성된 이미지 표시 (fade-in 애니메이션)
- error: "이미지 생성에 실패했어요. 다시 시도해 주세요." + 재시도 버튼

이미지 표시 스타일:
- border-radius: 16px
- 부드러운 box-shadow
- 이미지 아래에 작게 "AI가 그린 우리 가족 이야기" 캡션
- 다운로드 버튼 (이미지 저장)

디자인: 기존 서로ON 디자인 시스템 유지
(브랜드 블루 #2E5E8C, 앰버 #E0903C, Pretendard 폰트)
```

## 7단계 — Supabase 테이블 설정

```
Supabase stories 테이블에 이미지 관련 컬럼을 추가하는 SQL을 작성해줘.

추가할 컬럼:
- image_url TEXT           ← 생성된 이미지 URL
- image_scene TEXT         ← 추출된 Scene 문장 (재생성 시 참고용)
- image_generated_at TIMESTAMPTZ  ← 생성 시각

그리고 이미지 URL을 저장·조회하는 Supabase 헬퍼 함수도
/lib/supabase.ts 에 추가해줘:
- saveImageToStory(storyId, imageUrl, scene)
- getStoryImage(storyId)
```

## 8단계 — 비용·속도 최적화 (선택, 나중에)

```
이미지 생성 파이프라인에 아래 최적화를 추가해줘.

1. 캐싱: 같은 storyId의 이미지가 이미 있으면
   DALL-E를 다시 호출하지 말고 저장된 image_url 바로 반환.

2. 재생성: 사용자가 "다시 그리기" 버튼을 누르면
   기존 이미지를 덮어쓰고 새로 생성.

3. 에러 재시도: DALL-E API 실패 시 1회 자동 재시도,
   그래도 실패하면 에러 메시지.

4. 로딩 시간 안내: 이미지 생성은 보통 15~30초 소요.
   로딩 중 "보통 20초 정도 걸려요" 안내 문구 표시.
```

## 전체 흐름 요약 (참고)

```
[자녀가 "이미지 생성" 버튼 클릭]
        ↓
[/api/story-image POST 호출]
        ↓
[GPT-4o: 정리된 이야기 → Scene 한 문장 추출]
        ↓
[buildImagePrompt: 고정 스타일 블록 + Scene 합치기]
        ↓
[DALL-E 3: 이미지 생성 (~20초)]
        ↓
[Supabase: image_url 저장]
        ↓
[프론트: 이미지 표시 + 이야기와 함께 보기]
```

---

## 주의사항

- DALL-E 3는 이미지 1장당 약 **$0.04(standard)** 비용이 발생합니다. 캐싱(8단계)을 꼭 적용하세요.
- 생성 시간은 보통 **15~30초**입니다. 로딩 UX를 꼭 넣으세요.
- 이미지 URL은 OpenAI 서버에 **1시간만** 유효합니다. 생성 즉시 Supabase Storage에 복사해 저장하는 것을 권장합니다.
- DALL-E 3는 특정 스튜디오 이름(지브리 등)이 포함된 프롬프트를 거부합니다. 우리 프롬프트는 스튜디오 이름 없이 화풍을 묘사하므로 문제없습니다.
